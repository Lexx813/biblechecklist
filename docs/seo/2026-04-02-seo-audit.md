## Executive Summary

The technical foundation is solid — HTTPS, security headers, robots.txt, sitemap, SSR metadata, and AI crawler access are all above average. The site is in the top 10-15% of sites for GEO readiness. The main liabilities are:

1. Several pages that matter for conversion and trust (/pricing, /privacy, /terms, /contact) are SPA-only and invisible to Googlebot
2. Blog content is too thin (most posts under 400 words)
3. Schema gaps block rich results on 80+ static pages
4. www → non-www redirect is a temporary 307 (link equity not passing)

---

## CRITICAL Issues (fix immediately)

### C1 — www redirect is 307 Temporary, not 308 Permanent
**Impact:** All backlinks and external links pointing to www.nwtprogress.com pass zero PageRank to the canonical domain.  
**Fix:** In Vercel Dashboard → Domains → www.nwtprogress.com → set Redirect type = 308 Permanent. The `vercel.json` rule is being overridden by Vercel's platform-level domain handling.  
**Effort:** 5 minutes

### C2 — /pricing, /privacy, /terms, /contact are SPA-only (zero crawlable content)
**Impact:** Google sees these pages as empty or duplicate of the homepage. /pricing is a conversion page in the sitemap — Google will classify it as thin/duplicate. /privacy and /terms are trust signals for E-E-A-T on a YMYL-adjacent app handling user data.  
**Fix:** Create dedicated Next.js routes (`app/pricing/page.jsx`, `app/privacy/page.jsx`, etc.) with their own `metadata` exports, unique H1s, and server-rendered content. The pricing page should SSR the plan names, prices, and feature list (already exists in `WebApplication` schema — mirror it in visible HTML).  
**Effort:** 2-4 hours

### C3 — /study-topics index page is CSR-only (loads as skeleton)
**Impact:** The gateway page to all 8 theological Q&A pages is invisible to AI crawlers that don't execute JavaScript. Individual topic pages are SSR'd correctly.  
**Fix:** Convert `/app/study-topics/page.jsx` to use `generateStaticParams` or a server component that returns topic cards in the initial HTML payload.  
**Effort:** 1-2 hours

### C4 — Test post `/blog/2026-04-01-test-insight` is in the sitemap
**Impact:** Indexed test content dilutes content quality signals.  
**Fix:** Add `export const metadata = { robots: { index: false } }` to that post, or delete it from the database and add its slug to a noindex list.  
**Effort:** 15 minutes

## HIGH Issues (fix within 1 week)

### H1 — GA4 measurement ID has trailing newline in env var
**Impact:** Malformed script src URL (`?id=G-WVBXCJFGWQ\n`). Most browsers tolerate this but it may affect GA4 data fidelity in strict environments.  
**Fix:** In Vercel Dashboard → Settings → Environment Variables, delete and re-enter `NEXT_PUBLIC_GA_MEASUREMENT_ID` as exactly `G-WVBXCJFGWQ` (no trailing newline). Redeploy.  
**Effort:** 5 minutes

### H2 — Course schema missing `hasCourseInstance` on all /plans/[slug] pages
**Impact:** All reading plan pages have valid `Course` schema but earn zero Google rich results. Adding `hasCourseInstance` makes them eligible for Course rich results across 11 plan pages.  
**Fix:** In `app/plans/[slug]/page.jsx`, extend `schemaCourse` with:
```js
hasCourseInstance: {
  "@type": "CourseInstance",
  courseMode: "online",
  courseWorkload: `P${plan.totalDays}D`,
  instructor: { "@type": "Organization", "@id": "https://nwtprogress.com/#organization", name: "JW Study" }
}
```
**Effort:** 30 minutes

### H3 — Article schema on /study-topics and /books missing required fields
**Impact:** 66 book pages + 8 study topic pages = 74 static pages with `Article` schema that fails Google's rich result validator. Missing: `datePublished`, `author`, `image`, `publisher.logo`.  
**Fix:** Add these four fields to `schemaArticle` in both `app/study-topics/[slug]/page.jsx` and `app/books/[book]/page.jsx`:
```js
datePublished: "2025-11-01",
dateModified: "2026-01-01",
image: "https://nwtprogress.com/og-image.webp",
author: { "@type": "Organization", "@id": "https://nwtprogress.com/#organization", name: "JW Study" },
publisher: { "@type": "Organization", "@id": "https://nwtprogress.com/#organization", name: "JW Study",
  logo: { "@type": "ImageObject", url: "https://nwtprogress.com/icon-512.png", width: 512, height: 512 } }
```
**Effort:** 1 hour

### H4 — Blog posts too thin (most under 400 words, minimum is 1,500)
**Impact:** Posts on competitive queries ("How to Read the Entire Bible in a Year", "The Book of Isaiah", "Family Worship Night") cannot rank at current depth.  
**Priority posts to expand first:**
- "How to Read the Entire Bible in a Year Using the NWT" (393 words)
- "The Book of Isaiah: A Study Guide" (420 words)
- "Why I Started Reading the Bible Every Day" (348 words)
- "5 Tips for Building a Consistent Bible Reading Habit" (410 words)

**Effort:** 1-2 days of content writing

### H5 — Blog posts attributed to Organization, not Person
**Impact:** E-E-A-T requires author-level attribution for article content. First-person posts attributed to "JW Study" (Organization) weakens authoritativeness signals.  
**Fix:** In `app/blog/[slug]/page.jsx`, change the author schema to use `"@type": "Person"` with `name: post.profiles?.display_name` when available, falling back to Organization. Add an author byline block in the SSR fallback div.  
**Effort:** 1 hour

### H6 — Forum URLs in sitemap use raw UUIDs — no keyword slugs
**Impact:** 41 forum thread URLs follow `/forum/{uuid}/{uuid}` pattern — no keyword signal, no readability. If forum content is behind auth, these pages waste crawl budget and generate soft 404s.  
**Fix (auth-gated content):** Add `<meta name="robots" content="noindex">` to forum thread pages and remove them from `app/sitemap.js`.  
**Fix (public content):** Add slug fields to forum threads and update routes to use `/forum/{category-slug}/{thread-slug}`.  
**Effort:** 1-2 hours

### H7 — /about page only 246 crawlable words (minimum 500)
**Impact:** Below the minimum for an About page. The existing content (creator story, JW Library companion positioning, pricing transparency) is strong — just needs expansion.  
**Fix:** Expand to 500+ words by adding: creator background, multilingual support section, app roadmap transparency, community size/growth signal.  
**Effort:** 30-60 minutes

### H8 — No video content / YouTube presence
**Impact:** YouTube mention presence has ~0.737 correlation with AI citation frequency. The site has zero multi-modal content. A single "How to use JW Study" video would increase Perplexity/ChatGPT citation rates significantly.  
**Fix:** Record a 3-5 min screen recording, upload to YouTube, embed on the site, link from llms.txt.  
**Effort:** 1-2 days

## MEDIUM Issues (fix within 1 month)

### M1 — og:image missing on /blog, /forum, /plans, /about, /study-topics
**Fix:** Add `images: [{ url: 'https://nwtprogress.com/og-image.webp', width: 1200, height: 630 }]` to the `openGraph` and `twitter` metadata objects in each respective `page.jsx`.  
**Effort:** 30 minutes

### M2 — BlogPosting schema missing `mainEntityOfPage` and `publisher.logo`
**Fix:** Extend `schemaArticle` in `app/blog/[slug]/page.jsx` — add `mainEntityOfPage: { "@type": "WebPage", "@id": url }` and `publisher.logo` ImageObject.  
**Effort:** 30 minutes

### M3 — Organization schema has no `sameAs` links
**Impact:** No external identity corroboration for E-E-A-T.  
**Fix:** Add any live social/external profile URLs to `sameAs` array in `app/layout.jsx` Organization schema. Even a GitHub URL counts.  
**Effort:** 15 minutes

### M4 — Hreflang tags all point to the same URL
**Observed:** All 6 language variants (en, es, pt, fr, tl, zh) have `href="https://nwtprogress.com"` — the same URL. This is only valid for same-URL dynamic language switching (which is what the app does), but Google may not see language-specific content since switching is client-side.  
**Fix:** Either remove non-English hreflang entries (keep `x-default` only) until separate locale URLs are implemented, or implement `/es/`, `/pt/` etc. paths.  
**Effort:** 15 minutes (to remove), 1-2 weeks (to implement proper locale URLs)

### M5 — Sitemap `lastmod` on static pages uses `new Date()` (always looks freshly updated)
**Fix:** Replace `lastModified: new Date()` on stable pages (`/`, `/about`, `/privacy`, `/terms`) with their actual last-content-change date as a hardcoded ISO string.  
**Effort:** 15 minutes

### M6 — Blog posts and reading plans missing from llms.txt
**Fix:** Add `## Blog Posts` and `## Reading Plans` sections to `/public/llms.txt` with key URLs and one-line descriptions.  
**Effort:** 1-2 hours

### M7 — IndexNow not implemented
**Impact:** New blog posts and forum threads aren't instantly submitted to Bing/Yandex — must wait for scheduled recrawl.  
**Fix:** Generate a UUID key, place at `public/{key}.txt`, add to `robots.txt`, call IndexNow API from Supabase post-insert webhook on new blog/forum content.  
**Effort:** 2-3 hours

### M8 — All blog posts have same publication date range (March-April 2026)
**Impact:** 25+ posts with identical RSS pubDates reads as bulk-generated content to AI ranking systems.  
**Fix:** Update `created_at` timestamps in Supabase to reflect when content was actually written/finalized. Stagger them across 2-3 months.  
**Effort:** 30 minutes (SQL update)

### M9 — /blog index title not keyword-rich (just "Blog | JW Study")
**Fix:** Change to "Bible Study Blog for Jehovah's Witnesses | JW Study" and expand meta description to ~150 chars.  
**Effort:** 5 minutes

### M10 — No external source links on study topic pages
**Impact:** Each study topic cites Bible verses but no external academic or reference sources. The Michael/Christ blog post (which cites Darrell D. Hannah on archive.org) is the right model.  
**Fix:** Add 1-2 external reference links per study topic (JW.org for NWT text, a Bible dictionary, an academic source).  
**Effort:** 1-2 hours

### M11 — WebSite Sitelinks Searchbox `?q=` parameter may not be functional
**Fix:** Test whether the homepage responds to `?q=searchterm` with relevant content. If not, remove the `potentialAction` from the WebSite schema in `app/layout.jsx`.  
**Effort:** 15 minutes
